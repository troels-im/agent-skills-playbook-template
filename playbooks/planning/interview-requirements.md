# Interview Requirements

Use this playbook when a request is important but underspecified.

## Goal

Turn ambiguity into decisions, requirements, assumptions, and open questions
before creating tickets or implementation tasks.

## Workflow

1. Restate the goal and current uncertainty.
2. Ask only high-leverage questions.
3. Prefer questions that split the decision tree.
4. Read repository evidence before asking questions the repo can answer.
5. Capture decisions in the relevant work trace.
6. Stop when requirements, out-of-scope boundaries, and verification are clear
   enough to write a PRD, ADR, ticket, or task.

## Output

- decisions made
- assumptions
- open questions
- candidate requirements and acceptance criteria
- next playbook to run

## Anti-Patterns

Avoid:

- asking a long questionnaire before inspecting available context
- continuing the interview after the next action is clear
- treating guesses as decisions
