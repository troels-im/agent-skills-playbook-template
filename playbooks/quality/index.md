# Quality Playbooks

Use these playbooks for QA, health checks, and report-only validation.

## Read Order

1. Start at root `playbooks/index.md`.
2. Use this index to pick the smallest quality playbook.
3. Load only the selected playbook and references it names.

## Playbooks

- `health.md`: Run a language-neutral repository health pass.
- `qa-only.md`: Perform report-only QA without fixing.
- `qa.md`: QA and fix issues within accepted scope.
