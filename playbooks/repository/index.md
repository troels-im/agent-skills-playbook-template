# Repository Playbooks

Use these playbooks for repository governance, planning, reviews,
retrospectives, policies, and reusable workflow capture.

## Read Order

1. Start at root `playbooks/index.md`.
2. Use this index to pick the smallest repository playbook.
3. Load only the selected playbook and references it names.

## Playbooks

- `agent-skill-authoring.md`: Create small SKILL.md entrypoints that route into
  progressive docs and playbooks.
- `choose-workflow.md`: Route a user request to the smallest matching playbook
  or trace workflow.
- `evaluation-playbook.md`: Design evidence-rich product or workflow evals.
- `implementation-playbook.md`: Move from idea to prototype to production work
  without assuming a stack.
- `incident-retrospective.md`: Write evidence-based retrospectives.
- `no-matching-playbook.md`: Work safely when no existing playbook matches.
- `plan-issue.md`: Plan one small issue with checked-in trace artifacts.
- `playbook-specification.md`: Define playbook metadata, indexing,
  progressive loading, and lifecycle rules.
- `post-task-playbook-retrospective.md`: Feed task lessons back into playbooks.
- `repository-policy-change.md`: Add or change repository policy checks.
- `review-turn.md`: Run a subagent red-team, blue-team, judge review against
  explicit acceptance criteria.
- `setup-agent-playbooks.md`: Configure tracker, trace, docs, and review
  conventions for a consuming repository.
- `task-risk-classification.md`: Choose verification depth.
- `writing-playbooks-from-sessions.md`: Extract reusable playbooks from
  completed sessions.
